The product uses a modern and scalable architecture built on open-source technologies. Here's how it's structured:

1. Data Collection Layer

Tools like Filebeat and Metricbeat gather information from all computers and containers.

2. Processing Layer

Logstash organizes the data, and suspicious information is sent to VirusTotal for analysis.

3. Storage Layer

Elasticsearch stores all the data in a searchable format, like a library for logs and metrics.

4. Analysis and Alerting Layer

Kibana creates dashboards to visualize data, detect issues, and send alerts.

5. Isolation and Remediation Layer

LXC Containers isolate risky activities in a sandbox, and automated scripts fix problems quickly.

        choco install vcxsrv

2. Ansible Playbook for LXC Setup


        ---
        - name: Install LXC and Chrome in Ubuntu container
          hosts: all
          tasks:
            - name: Install LXC
              apt:
                name: lxd
                state: present
        
            - name: Create LXC container
              command: lxc launch ubuntu:20.04 chrome-container
        
            - name: Install Chrome inside container
              command: lxc exec chrome-container -- apt install google-chrome-stable -y
        
            - name: Install Filebeat and Metricbeat
              apt:
                name: "{{ item }}"
                state: present
              loop:
                - filebeat
                - metricbeat

3. Automating VirusTotal Checks


        #!/bin/bash
        # Sample script to check URLs with VirusTotal API
        
        API_KEY="your_api_key_here"
        URL=$1
        
        # Encode the URL to base64
        url_id=$(echo -n "$URL" | base64 | tr -d '\n' | tr '/+' '_-' | tr -d '=')
        
        # Make the VirusTotal API call
        response=$(curl -s --request GET \
          --url "https://www.virustotal.com/api/v3/urls/$url_id" \
          --header "x-apikey: $API_KEY")
        
        # Output the result
        echo "$response"

Remediation & Actionable Steps

üõ°Ô∏è Remediation & Actionable Steps

Steps to take when an infected URL or file is detected:

  • Isolate Compromised Systems: Use Ansible or Elastic Agent to isolate machines or containers that have been infected by malicious URLs or files.
  • Quarantine URLs and Files: Add infected URLs and files to a quarantine list for further investigation. This list should be stored in the centralized database.
  • Conduct Post-Incident Analysis: Use Kibana to generate reports and analyze the scope of the attack (e.g., how many users clicked infected URLs).
