CLASSIFICATION: CRITICAL / CONFIRMED EXPOSURE

Treat as a probable compromise

An unauthenticated automation hub on the public internet must be assumed reachable by everyone, for the entire time it was up. You cannot prove a negative from the outside — so the working assumption is breach, and the first job is to stop destroying the evidence that tells you what actually happened.

// 01 — The golden hour

Don't destroy the crime scene

The instinct is to "just reboot it and patch." That instinct wipes the only proof of what an attacker did. Before anything else — freeze.

⛔

Do not power off / reboot

A reboot destroys volatile memory — running processes, in-RAM payloads, live network sessions. That's often where the only evidence lives.

🗑️

Do not delete or re-pull

Don't docker rm the container or pull a fresh image. You'd erase attacker-modified files and the timeline of changes.

🩹

Do not "quietly fix"

Patching in place before imaging contaminates the scene and can tip off an attacker who's watching. Preserve first, fix second.

▸ Collect in order of volatility — most fragile first
1Memory (RAM)running processes, injected code, decrypted secrets, live sessions
2Network stateactive connections, listening ports, current egress destinations
3Containerssnapshot the live container before any change touches it
4Disk & configsn8n data dir, database, env / config, reverse-proxy setup
5Logsapp, proxy access/error, host auth — copy, don't rotate away

// 03 — What to hunt for

Indicators & anomalies

In an exposed n8n, these are the tells that someone did more than look.

🌍

Foreign hits on /rest/ /webhook/

Successful (2xx) requests to API or webhook paths from IPs / geographies that don't match your team or known integrations.

🧬

New or altered workflows

Workflows you don't recognise, recently modified timestamps, or Code nodes containing long, high-entropy / base64 blobs.

🔑

Unknown credentials

Credential entries nobody created, or connections pointing at endpoints you don't operate — a sign of attacker-added access.

📤

Unexpected egress

Outbound connections to unfamiliar hosts, or scheduled flows suddenly calling external URLs / paste sites.

🧱

Changed container files

Binaries, cron entries or scripts added inside the container since deploy — surfaced by a file diff against the base image.

⏱️

Off-hours execution

Manual workflow runs or logins at times that don't fit normal operations — the quiet hours an intruder prefers.
