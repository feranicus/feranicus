04 / Safety by design

Two models, one guardrail

Healthcare needs strict safety boundaries, so conversation is separated from governance. A fast model talks; a high-reasoning model audits every clinical claim out-of-band before it ever reaches the patient or the database.

CONVERSATION

Qwen 3.6

qwen-3.6-instruct · via OpenRouter

Handles multi-turn dialogue, Hebrew↔English translation, and wellness metrics. Never claims to be a doctor or issues prescriptions; always frames natural remedies as adjunctive; escalates serious symptoms to standard screening. Runs read-only — zero write access to the database.

⟶
AUDIT · COWORK

Claude Opus 4.8

out-of-band clinical supervisor

Every medical recommendation, supplement dose, and lab interpretation is routed to Claude Opus 4.8 (Cowork) before delivery. If it detects a safety violation — advising someone to stop a medication, or an incorrect dose — it halts delivery, logs a warning, and drafts a safer alternative.

07 / LATAM → US

The arbitrage pipeline

The economic engine of Pillar 3: bridging US patients to high-quality bioequivalent generics under the FDA Personal Importation Policy, with logistics structured for legal insulation.

🧭 FDA Personal Importation (PIP) gate

1Unavailability / cost barrier — drug is unavailable domestically or prohibitively priced, qualifying as a barrier to continued treatment.
2≤ 90-day supply — each import occurrence is limited to a 3-month personal-use quantity.
3Valid prescription — authentic Rx from a licensed practitioner, or attested ongoing clinical supervision.
4Safety filter — narcotics and highly-restricted controlled substances are programmatically filtered out of the arbitrage engine.

09 / Secure-by-design

Zero-trust & PII-safe

Aligned to CISA (US) and CSA Singapore secure-by-design frameworks, plus HIPAA Security Rule and PDPA data-handling. Three load-bearing controls:

🧱

Execution firewall

Qwen 3.6 runs in a read-only sandbox with zero write access to the primary database. Any output with dosages or lab interpretation must pass the Claude Opus 4.8 validation schema.

🕸️

Network segmentation

Redis and PostgreSQL sit on an internal: true Docker network — no route to or from the public internet. Secrets load from an external .env, never hardcoded, excluded from version control.

🫥

PII scrubber pipeline

Hermes strips phones, emails, national IDs and NRICs from every payload before it reaches any LLM API — satisfying PDPA and international health-privacy law.

# Hermes PII scrubber — runs before any payload leaves for an LLM
class PIIScrubber:
    def scrub_text(self, text):
        text = self.phone_pattern.sub("[MASKED_PHONE]", text)
        text = self.email_pattern.sub("[MASKED_EMAIL]", text)
        text = self.israel_id_pattern.sub("[MASKED_NATIONAL_ID]", text)
        text = self.singapore_nric.sub("[MASKED_NRIC]", text)
        return text

# "my phone is 054-1234567, email ester@gmail.com"
# → "my phone is [MASKED_PHONE], email [MASKED_EMAIL]"
10 / Growth on autopilot

Autonomous marketing loop

A cron worker inside Hermes closes the loop continuously, minimizing customer-acquisition cost without human overhead — always FDA/FTC structure-function compliant.

1Ingest — poll Meta ad dashboards & landing-page analytics.
2Trigger — if CTR < 1.5%, a refresh run begins.
3Generate — Claude drafts 5 compliant copy variants for the weak segment.
4Assets — image models produce on-brand banners via strict style schemas.
5Compile & A/B — static landing variants on edge (S3 / Cloudflare Workers).
6Dispatch — new ads pushed via platform APIs, launched PAUSED for human verification.
